# Contributing to Ghost in M'Sheet

Thank you for your interest in contributing to Ghost in M'Sheet, a community-driven continuation of Trykowka's "The Ghost Hunter"!

This project thrives on community involvement. Anyone can suggest ideas, report bugs, or propose story content. Admins review and approve all suggestions before they are added to the roadmap or worked on.

---

## Ways to Contribute

### Suggest a Feature

Have an idea for a new mechanic, ghost type, location, or gameplay system? Open a **Feature Request** issue.

### Report a Bug

Found something broken or behaving unexpectedly? Open a **Bug Report** issue.

### Propose a Plotline or Story Content

Want to suggest a new questline, character arc, story event, or scene? Open a **Plotline / Story Suggestion** issue.

### General Discussion

Have broader ideas, questions, or want to chat with the community? Head to the [Discussions](../../discussions) tab.

---

## How the Suggestion to Roadmap Pipeline Works

1. **You open an issue** using one of the templates (feature, bug, plotline, etc.)
2. **Admins review** your suggestion -- they may ask clarifying questions in the comments
3. **If approved**, admins add it to the [project roadmap](../../projects/1) with an appropriate status
4. **The community can upvote** (thumbs-up reaction) issues to signal priority
5. **Work begins** when a contributor picks it up or an admin assigns it

> Note: Suggestions are not guaranteed to be implemented. Admins have final say on what goes on the roadmap.

---

## Submitting Code or Content (Pull Requests)

If you want to contribute code or Twee/story content directly:

1. **Fork** the repository
2. Create a branch: `git checkout -b my-feature-name`
3. Make your changes -- see [BUILD_INSTRUCTIONS.md](BUILD_INSTRUCTIONS.md) and [SETUP.md](SETUP.md) for how to build the project
4. **Test** your build with `./build.sh` before submitting
5. Open a **Pull Request** against `main` with a clear description of what you changed and why

### PR Guidelines

- Keep PRs focused -- one feature or fix per PR
- Reference the related issue (e.g. `Fixes #12`) in your PR description
- For story/plotline content, include a brief summary of the narrative and how it fits the game
- Asset contributions should follow the existing pattern in `asset-placeholders/`

---

## Issue Labels

| Label | Meaning |
|---|---|
| `feature` | New mechanic, system, or gameplay element |
| `bugfix` | Something that is broken or incorrect |
| `plotline` | Story, quest, or narrative content |
| `ghost` | New ghost type or ghost-related content |
| `location` | New in-game location |
| `character` | New or expanded NPC/companion |
| `balance` | Tuning of existing mechanics |
| `refactor` | Code quality / technical debt |
| `needs-review` | Awaiting admin review |
| `approved` | Approved for the roadmap |
| `wontfix` | Will not be implemented |

---

## Community Guidelines

- Be respectful and constructive in all discussions
- Search existing issues before opening a new one -- your idea may already be tracked
- Provide as much detail as possible in your suggestions so admins can evaluate them fairly
- This is an adult game -- suggestions for content should be clearly described in the issue template

---

## Questions?

If you're unsure where to start, open a [Discussion](../../discussions) and we'll help you out.
